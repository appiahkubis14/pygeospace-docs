pygeospace Documentation

A small, dependency-light geospatial visualization toolkit for Python. Read GeoJSON, attach styled vector layers to a map, and export a standalone interactive HTML map you can open in any browser — no build step, no server.

Getting Started

Up and running in 2 minutes

The core install has a single dependency (Click, for the CLI) and reads GeoJSON out of the box. Build a map in a few lines of Python, or straight from the command line.

1
Install
# Core — GeoJSON + HTML export pip install pygeospace # + Shapefile / GeoPackage / KML / GPX pip install "pygeospace[formats]"
2
Verify
pygeospace --version # → pygeospace, version 0.1.0 pygeospace --help # lists: visualize, info, version
3
Build a map (Python)
import pygeospace as pgs layer = pgs.read("cities.geojson", color="#e53e3e", radius=9) m = pgs.Map(center=(0, 25), zoom=2, basemap="carto-light") m.add_layer(layer) m.save("map.html")
4
…or from the CLI
pygeospace visualize cities.geojson \ --output map.html \ --color "#e53e3e" \ --radius 9 \ --basemap carto-light # Wrote map.html (N features).
What you get: map.html is fully standalone — open it in any browser for a pan / zoom / click map. MapLibre GL is loaded from a CDN at view time; there is no server and no build step.

Core Concepts

How pygeospace fits together

Three objects do the work: a Map holds a view and a stack of layers, a VectorLayer wraps styled GeoJSON, and read() loads data into a layer. The map serializes itself to a self-contained HTML file.

🗺️
Map
A view state (center, zoom, basemap) plus an ordered list of layers. Knows how to serialize to a dict and render to standalone HTML.
📍
VectorLayer
Wraps a GeoJSON FeatureCollection with a small style dict. Points, lines and polygons each pick the right draw type automatically.
📥
read()
Loads a file into a VectorLayer. GeoJSON needs nothing extra; other vector formats route through GeoPandas when installed.

Layers

Layer is the abstract base. VectorLayer is the implemented subclass today. Raster, point-cloud and streaming layers are on the roadmap and are intentionally not shipped yet — calls into those paths raise a clear error rather than silently doing nothing.

LayerStatusBacked by
VectorLayer✓ ImplementedGeoJSON FeatureCollection
RasterLayer○ PlannedCOG via tile server (roadmap)
PointCloudLayer○ PlannedLAS/LAZ, 3D Tiles (roadmap)
StreamLayer○ PlannedWebSocket source (roadmap)

Reading data

read() auto-detects by extension. GeoJSON is parsed with the standard library. The formats below require the optional [formats] extra (GeoPandas); without it, read() raises NotImplementedError with an install hint instead of failing obscurely.

.geojson / .json
core · no deps
Parsed directly. A single Feature is wrapped into a FeatureCollection.
.shp
[formats]
Shapefile via GeoPandas / Fiona.
.gpkg
[formats]
GeoPackage via GeoPandas.
.kml / .gpx
[formats]
KML and GPX tracks via GeoPandas.
.fgb / .gml
[formats]
FlatGeobuf and GML via GeoPandas.

Styling

Every VectorLayer carries a style dict merged over sensible defaults. Pass overrides to read() / VectorLayer(...), or call set_style(**kwargs). Colors are plain CSS strings.

KeyDefaultApplies toDescription
color#2b6cb0allFill / line / circle color.
opacity0.8allLayer opacity, 0–1.
radius6pointsCircle radius in pixels.
stroke_color#1a365dpoints / polygonsOutline color.
stroke_width1allOutline / line width in pixels.

Basemaps

Three raster basemaps ship built in. Pass one to Map(basemap=...). (Each pulls standard XYZ tiles; attribution is included in the exported map.)

carto-light
default
CARTO Positron — clean, light, good under colorful data.
carto-dark
dark
CARTO Dark Matter — high-contrast dark canvas.
osm
detailed
OpenStreetMap standard — full streets and labels.

HTML export

to_html() returns the map as an HTML string; save(path) writes it to disk and returns the resolved path. The output embeds your data as GeoJSON and wires up MapLibre GL: navigation controls, hover cursors, and click-to-inspect popups showing each feature's properties.

No browser storage: the exported page keeps all state in memory — it never uses localStorage or a backend. Drop it on static hosting (GitHub Pages, S3) and it just works.
Python API

API Reference

The public surface is deliberately small: Map, VectorLayer (and its base Layer), and read. Everything is importable from the top-level package.

imports
import pygeospace as pgs
from pygeospace import Map, VectorLayer, Layer, read

Map

Mapclass
Map(center=(0.0, 0.0), zoom=2, basemap="carto-light")
Create a 2D web map. Raises ValueError for an unknown basemap.
center
tuple[float,float]
(longitude, latitude) of the initial view. Longitude first.
zoom
float
Initial zoom. 0 = whole world, ~18 = building scale.
basemap
str
"carto-light", "carto-dark", or "osm".
add_layer(layer: Layer) → Map
Add a layer to the top of the stack. Returns self for chaining. Raises ValueError on a duplicate layer name.
layers → list[Layer] (property)
A copy of the current layer stack.
to_dict() → dict
Serialize the full map state (center, zoom, resolved basemap URL, layer specs) to a JSON-able dict.
to_html() → str
Render the map to a standalone HTML string (MapLibre GL).
save(path: str | Path) → Path
Write the map to an HTML file and return the resolved path.
Map.BASEMAPS → dict[str, str] (class attribute)
Mapping of basemap keys to XYZ tile URL templates.

VectorLayer

VectorLayerdataclass
VectorLayer(name: str, visible=True, geojson={...}, style={})
A styled vector layer backed by a GeoJSON FeatureCollection. The supplied style is merged over the defaults documented above.
name
str
Layer identifier; must be unique within a Map.
geojson
dict
A GeoJSON FeatureCollection. Points, lines and polygons all supported.
style
dict
Style overrides (see Styling).
visible
bool
Whether the layer renders. Default True.
feature_count → int (property)
Number of features in the layer.
set_style(**kwargs) → VectorLayer
Update style properties in place. Returns self for chaining.
filter(predicate) → VectorLayer
Return a new layer with only the features whose properties satisfy predicate(props). The original is untouched.
to_spec() → dict
Serialize to the renderer spec embedded in exported HTML.

read()

readfunction
read(path: str | Path, name=None, **style) → VectorLayer
Read a vector dataset into a VectorLayer. name defaults to the file stem. Extra keyword args become style overrides. Raises NotImplementedError for formats needing GeoPandas when it is not installed, or for unsupported extensions.
full_workflow.py
import pygeospace as pgs

# Read GeoJSON and style it
cities = pgs.read("cities.geojson", color="#e53e3e", radius=9)

# Derive a filtered layer (original is unchanged)
big = cities.filter(lambda p: p["population"] > 8_000_000)
big.name = "megacities"
big.set_style(color="#1da3d8", radius=14)

# Compose a map and export
m = pgs.Map(center=(0, 25), zoom=1.5, basemap="carto-light")
m.add_layer(cities)
m.add_layer(big)
out = m.save("map.html")
print(f"Wrote {out} — {cities.feature_count} cities, {big.feature_count} megacities")

Examples

Copy-paste recipes

Each example is self-contained and runs against the current release.

A point map from inline data

cities.py
import pygeospace as pgs

cities = {
    "type": "FeatureCollection",
    "features": [
        {"type": "Feature",
         "geometry": {"type": "Point", "coordinates": [-0.13, 51.51]},
         "properties": {"city": "London", "population": 8982000}},
        {"type": "Feature",
         "geometry": {"type": "Point", "coordinates": [-0.19, 5.60]},
         "properties": {"city": "Accra", "population": 2388000}},
        {"type": "Feature",
         "geometry": {"type": "Point", "coordinates": [139.69, 35.69]},
         "properties": {"city": "Tokyo", "population": 13960000}},
    ],
}

layer = pgs.VectorLayer(name="cities", geojson=cities,
                        style={"color": "#e53e3e", "radius": 9})
pgs.Map(center=(0, 25), zoom=1.5).add_layer(layer).save("cities.html")

Filtering features into a second layer

filter.py
import pygeospace as pgs

cities = pgs.read("cities.geojson")
mega = cities.filter(lambda p: p["population"] > 8_000_000)
mega.name = "megacities"
mega.set_style(color="#1da3d8", radius=14)

m = pgs.Map(center=(0, 25), zoom=1.5)
m.add_layer(cities)   # all cities, default blue
m.add_layer(mega)     # big ones, emphasized on top
m.save("filtered.html")

Reading a Shapefile or GeoPackage

Requires pip install "pygeospace[formats]".

from_file.py
import pygeospace as pgs

# Any GeoPandas-readable vector format
regions = pgs.read("regions.gpkg", color="#059669", opacity=0.5)
pgs.Map(center=(0, 25), zoom=4, basemap="osm") \
   .add_layer(regions) \
   .save("regions.html")
Reference

Current Scope — what's in, what's not

pygeospace is an early foundation. This section is deliberately explicit so you can tell, at a glance, whether it fits your use case today.

✅ In v0.1.0
  • Read GeoJSON (stdlib) and GeoPandas formats (optional extra)
  • Styled vector layers: points, lines, polygons
  • Immutable filter() and chainable set_style()
  • Three built-in basemaps
  • Standalone interactive HTML export (MapLibre GL)
  • Click popups + navigation controls in output
  • A small CLI: visualize, info, version
○ Not built yet
  • Raster / COG layers and a tile server
  • Point-cloud and 3D / globe rendering
  • Real-time / streaming layers
  • On-the-fly analytics (NDVI, clustering, networks)
  • Choropleth classification + legends
  • WebGPU / vector-tile performance paths
  • Static image (PNG/PDF/SVG) export
Design principle: unbuilt features fail loudly. Reading an unsupported format, for instance, raises NotImplementedError naming exactly what to install or which formats are supported — never a silent no-op.

A note on scale claims: "billions of points" in a browser is achieved through tiling and aggregation, not by rendering every feature. Performance targets here will be stated as frames-per-second on named hardware against fixed datasets — measured, never asserted.
Community

Contributing

The codebase is small and approachable — a good place to add a feature end to end. The most useful next contributions are the v0.2.0 items above.

dev setup
# Clone and install in editable mode with dev + formats extras
git clone https://github.com/your-org/pygeospace
cd pygeospace
pip install -e ".[dev,formats]"

# Run the tests
pytest          # or: PYTHONPATH=. python tests/test_core.py

# Build distribution artifacts
python -m build
twine check dist/*
Before you publish: pygeospace may be taken on PyPI. Pick a unique distribution name and verify it with curl -s -o /dev/null -w "%{http_code}" https://pypi.org/pypi/<name>/json (404 = available). Keep the package description truthful to what the code actually does.
